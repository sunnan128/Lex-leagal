# LexAI 多 Agent 架构升级路线图

> 面试讲述定位：**"从单服务 RAG 演进到多智能体架构的系统优化过程"**
> 每一步都是上一步遇到问题后的自然演进，不是为改而改。

---

## 当前架构（V2 — 已完成）

```
用户提问 → QAService.query() → 检索 → Rerank → LLM生成 → 返回
```

**问题**：所有逻辑串行耦合在一个方法里，无法做意图分流、输出审查、链路回溯。

---

## 升级总览

```
V2（当前）         V3                    V4                    V5
单服务 RAG  ──→  多 Agent 编排  ──→  上下文记忆  ──→  评估体系
                （本次重点）           （后续可选）     （后续可选）
```

## 第一阶段：多 Agent 架构改造（V3）

### 目标

将 `QAService.query()` 拆分为 4 个 Agent，用 LangGraph 编排。

### 架构图

```
用户提问
    │
    ▼
┌─────────────┐     非法律问题      ┌──────────────────────────┐
│  路由 Agent  │ ─────────────────→ │ 直接回复：                 │
│  (Router)   │                     │ "我是法律问答助手，        │
└──────┬──────┘                     │  请提问法律相关问题"       │
       │ 法律问答意图                └──────────────────────────┘
       ▼
┌─────────────┐
│  检索 Agent  │   混合检索 + Rerank（复用现有代码）
│  (Retriever)│
└──────┬──────┘
       │ Top-K 检索结果
       ▼
┌─────────────┐
│  生成 Agent  │   组装 Context + 调用 LLM（复用现有代码）
│  (Generator) │
└──────┬──────┘
       │ 回答 + 溯源
       ▼
┌─────────────┐      不通过        ┌─────────────┐
│  审查 Agent  │ ────────────────→ │  打回路由     │（最多重试 2 次）
│  (Reviewer) │                    └─────────────┘
└──────┬──────┘
       │ 通过
       ▼
    返回用户
```

### 步骤 1.1：新建 Agent 目录和状态定义

**新建文件**：`backend/agents/__init__.py`

**新建文件**：`backend/agents/state.py` — 定义 Agent 间传递的状态

```python
from typing import TypedDict, List, Dict, Any, Optional, Literal

class AgentState(TypedDict):
    """Agent 间传递的共享状态"""
    # 输入
    question: str
    top_k: int
    use_rerank: bool
    use_keyword_search: bool

    # 路由
    intent: Optional[Literal["legal_query", "chitchat"]]

    # 检索
    search_results: List[Dict[str, Any]]

    # 生成
    answer: str
    citations: List[Dict[str, Any]]
    found_in_knowledge_base: bool

    # 审查
    review_passed: bool
    review_reason: str

    # 控制
    retry_count: int
    processing_time_ms: float
```

**为什么要 TypedDict 而不是全局变量**：状态在 Agent 间显式传递，链路清晰可追溯，方便调试。

### 步骤 1.2：实现路由 Agent

**新建文件**：`backend/agents/router_agent.py`

**职责**：判断是否为法律问题，非法律问题直接回复引导。

**实现思路**：
```python
import re

class RouterAgent:
    def __init__(self, llm_service):
        self.llm = llm_service

    def route(self, state: dict) -> dict:
        question = state["question"]

        # 第一层：规则匹配（零延迟，覆盖高频场景）
        intent = self._rule_match(question)

        # 第二层：LLM 分类（规则未命中时才调用，节省 token）
        if intent is None:
            intent = self._llm_classify(question)

        state["intent"] = intent

        # 非法律问题直接给出引导回复，不走检索
        if intent == "chitchat":
            state["answer"] = "我是法律文档智能问答助手，请提问法律相关问题。"
            state["found_in_knowledge_base"] = False
            state["review_passed"] = True  # 引导回复无需审查

        return state

    def _rule_match(self, question: str) -> str | None:
        """规则优先，零 token 消耗"""
        # 包含"第X条/款/章"的法律问题 → 直接判定
        if re.search(r'第[\d零一二三四五六七八九十百千]+[条款项章节编]', question):
            return "legal_query"
        # 包含法律相关关键词
        legal_keywords = ["刑法", "民法", "合同", "侵权", "违约", "赔偿",
                         "定罪", "量刑", "缓刑", "有期徒刑", "罚金",
                         "劳动者", "用人单位", "劳动合同", "工伤",
                         "消费者", "权益", "退赔", "抵押"]
        if any(kw in question for kw in legal_keywords):
            return "legal_query"
        return None  # 规则未命中，交给 LLM

    def _llm_classify(self, question: str) -> str:
        """LLM 意图分类（规则未命中时才走到这里）"""
        prompt = f"""判断以下问题是否属于法律知识问答，只输出"legal_query"或"chitchat"：

用户问题：{question}

只输出类别名称。"""

        response = self.llm.client.chat.completions.create(
            model=self.llm.model,
            messages=[{"role": "user", "content": prompt}],
            temperature=0.0,
            max_tokens=20
        )
        result = response.choices[0].message.content.strip().lower()
        if "legal" in result:
            return "legal_query"
        return "chitchat"
```

**设计要点**：
- **两层策略**：规则优先（零 token），规则未命中才调 LLM
- **法律关键词库**：覆盖刑法、民法、劳动法、消费者权益等常见领域
- **非法律问题直接回复**：不浪费检索和生成资源

### 步骤 1.3：实现检索 Agent

**新建文件**：`backend/agents/retriever_agent.py`

**职责**：执行混合检索 + Rerank。

**实现思路**：直接复用 `vector_store.py` 和 `llm_service.py` 的现有代码，只是封装成 Agent 类。

```python
class RetrieverAgent:
    def __init__(self, vector_store, llm_service, config):
        self.vector_store = vector_store
        self.llm_service = llm_service
        self.config = config

    def retrieve(self, state: dict) -> dict:
        question = state["question"]

        if state["use_rerank"]:
            # 混合检索取 top-20 → Rerank 精排 → top-5
            candidates = self.vector_store.hybrid_search(
                question,
                top_k=state["top_k"],
                rerank_candidates=self.config.RERANK_CANDIDATES
            )
            search_results = self.llm_service.rerank_results(
                question, candidates, state["top_k"]
            )
        elif state["use_keyword_search"]:
            search_results = self.vector_store.hybrid_search(
                question, state["top_k"]
            )
        else:
            search_results = self.vector_store.semantic_search(
                question, state["top_k"]
            )

        state["search_results"] = search_results
        return state
```

**关键**：检索逻辑一行不改，只是包了一层 Agent 接口。

### 步骤 1.4：实现生成 Agent

**新建文件**：`backend/agents/generator_agent.py`

**职责**：基于检索结果生成回答 + 构建溯源。

**实现思路**：复用 `llm_service.py` 的 `generate_answer()` 和 `_build_context()`。

```python
class GeneratorAgent:
    def __init__(self, llm_service):
        self.llm_service = llm_service

    def generate(self, state: dict) -> dict:
        answer, citations, found_kb, _ = self.llm_service.generate_answer(
            state["question"],
            state["search_results"],
            state["use_rerank"]
        )
        state["answer"] = answer
        state["citations"] = [c.model_dump() for c in citations]
        state["found_in_knowledge_base"] = found_kb
        return state
```

### 步骤 1.5：实现审查 Agent（核心亮点，全新代码）

**新建文件**：`backend/agents/reviewer_agent.py`

**职责**：检查生成结果是否可靠，不通过则打回重试。

**实现思路**：规则检查为主，不调用 LLM（零 token 消耗）。

```python
class ReviewerAgent:
    def __init__(self):
        pass

    def review(self, state: dict) -> dict:
        answer = state["answer"]
        search_results = state["search_results"]

        # 检查 1：是否为空回答
        if not answer or len(answer.strip()) < 10:
            state["review_passed"] = False
            state["review_reason"] = "回答内容过短"
            return state

        # 检查 2：检索为空但模型仍生成了内容（幻觉）
        if not search_results and state["found_in_knowledge_base"]:
            state["review_passed"] = False
            state["review_reason"] = "检索结果为空但模型生成了内容，疑似幻觉"
            return state

        # 检查 3：回答是否引用了检索结果
        if search_results:
            has_reference = False
            for result in search_results:
                # 取检索结果的前 20 个字符作为引用标记
                ref_snippet = result['content'][:20]
                if ref_snippet in answer:
                    has_reference = True
                    break
            # 如果回答中有 [文档X] 标记也算有引用
            if "[文档" in answer:
                has_reference = True

            if not has_reference and len(answer) > 100:
                state["review_passed"] = False
                state["review_reason"] = "回答未引用任何检索结果，可能存在幻觉"
                return state

        # 检查 4：回答是否包含"知识库中未找到"但 found_kb 为 True（矛盾）
        if "知识库中未找到" in answer and state["found_in_knowledge_base"]:
            state["review_passed"] = False
            state["review_reason"] = "回答声明未找到但检索有结果，判定矛盾"
            return state

        # 全部通过
        state["review_passed"] = True
        state["review_reason"] = "审查通过"
        return state
```

**为什么不调 LLM**：规则检查毫秒级完成，零 token，覆盖 90% 的幻觉场景。面试时说"审查用规则为主，不增加 API 成本"。

### 步骤 1.6：用 LangGraph 编排

**新建文件**：`backend/agents/graph.py`

**职责**：定义 Agent 执行流程图。

```python
from langgraph.graph import StateGraph, END
from backend.agents.state import AgentState
from backend.agents.router_agent import RouterAgent
from backend.agents.retriever_agent import RetrieverAgent
from backend.agents.generator_agent import GeneratorAgent
from backend.agents.reviewer_agent import ReviewerAgent

MAX_RETRIES = 2

def build_graph(vector_store, llm_service, config):
    router = RouterAgent(llm_service)
    retriever = RetrieverAgent(vector_store, llm_service, config)
    generator = GeneratorAgent(llm_service)
    reviewer = ReviewerAgent()

    graph = StateGraph(AgentState)

    # 添加节点
    graph.add_node("router", router.route)
    graph.add_node("retriever", retriever.retrieve)
    graph.add_node("generator", generator.generate)
    graph.add_node("reviewer", reviewer.review)

    # 设置入口
    graph.set_entry_point("router")

    # 路由 → 根据意图分流
    graph.add_conditional_edges(
        "router",
        lambda state: state["intent"],
        {
            "legal_query": "retriever",  # 法律问答走检索链路
            "chitchat": END              # 非法律问题直接返回引导回复
        }
    )

    # 检索 → 生成 → 审查
    graph.add_edge("retriever", "generator")
    graph.add_edge("generator", "reviewer")

    # 审查 → 通过/打回
    graph.add_conditional_edges(
        "reviewer",
        lambda state: "retry" if (
            not state["review_passed"]
            and state["retry_count"] < MAX_RETRIES
        ) else "end",
        {
            "retry": "router",  # 打回路由，重新走完整链路
            "end": END          # 通过或超过重试次数，结束
        }
    )

    return graph.compile()
```

**为什么打回路由而不是原地重试**：可能是路由本身分错了，重新走完整链路更可靠。

### 步骤 1.7：修改 QAService 对接 Agent 图

**修改文件**：`backend/services/qa_service.py`

```python
# 在 QAService.__init__ 中初始化 Agent 图
from backend.agents.graph import build_graph

class QAService:
    def __init__(self):
        self.vector_store = VectorStoreService()
        self.llm_service = LLMService()
        self.agent_graph = build_graph(self.vector_store, self.llm_service, settings)
        # ... 其他不变

    async def query(self, request: QueryRequest) -> QueryResponse:
        import time
        start_time = time.time()

        # 构建初始状态
        initial_state = {
            "question": request.question,
            "top_k": request.top_k,
            "use_rerank": request.use_rerank,
            "use_keyword_search": request.use_keyword_search,
            "intent": None,
            "search_results": [],
            "answer": "",
            "citations": [],
            "found_in_knowledge_base": False,
            "review_passed": False,
            "review_reason": "",
            "retry_count": 0,
            "processing_time_ms": 0.0
        }

        # 执行 Agent 图
        final_state = self.agent_graph.invoke(initial_state)

        processing_time = (time.time() - start_time) * 1000

        # 构建 citations
        citations = [
            Citation(**c) for c in final_state.get("citations", [])
        ]

        return QueryResponse(
            answer=final_state["answer"],
            citations=citations,
            found_in_knowledge_base=final_state["found_in_knowledge_base"],
            processing_time_ms=processing_time
        )
```

## 面试讲述版本演进

> **V1**（初始版）：纯向量检索，单服务串行调用
>
> **V2**（当前）：加了混合检索（BM25 + 向量）+ Rerank，精准率从 68% 提到 91%
>
> **V3**（本次升级）：从单服务演进到多 Agent 架构，用 LangGraph 编排 4 个 Agent
> - 为什么要拆：原来 query() 里检索、生成、返回耦合在一起，无法做意图分流和输出审查
> - 拆了什么：路由 → 检索 → 生成 → 审查
> - 路由怎么做的：规则优先匹配法律关键词（零 token），未命中再调 LLM 分类
> - 审查做了什么：规则检查幻觉、引用验证、矛盾检测，不通过打回重试，最多 2 次
> - 对性能的影响：路由大部分场景规则直接命中，审查用规则不调 LLM，token 成本基本不增加

## 后续可选升级（V4 / V5）

在 `AgentState` 中增加 `conversation_history: List[Dict]`，生成 Agent 在构建 prompt 时拼接历史对话，支持追问如"刚才那条再说详细点"。
