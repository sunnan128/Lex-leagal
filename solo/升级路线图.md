## 第一阶段：评估体系 + 多轮对话（V3）

### 目标

**用数据证明效果好，不是"我觉得"**，同时补全 RAG 系统的基础能力。

### 步骤 1.1：接入 RAGAS 评估框架

**新建文件**：`backend/evaluation/ragas_eval.py`

**做什么**：自动评估 RAG 系统的四项核心指标。

| 指标 | 含义 | 为什么重要 |
|------|------|-----------|
| **faithfulness** | 回答是否忠实于检索到的文档 | 衡量幻觉程度 |
| **answer_relevancy** | 回答是否与问题相关 | 衡量答非所问 |
| **context_precision** | 检索到的文档是否精确 | 衡量检索质量 |
| **context_recall** | 是否检索到了所有相关内容 | 衡量召回完整度 |

**怎么做**：
```python
# 1. 准备测试集（50-100 个问答对）
# 格式：[{"question": "...", "ground_truth": "..."}]

# 2. 对每个问题走完整 RAG 链路，记录检索结果和生成回答

# 3. 用 RAGAS 自动评分
from ragas import evaluate
from datasets import Dataset

eval_dataset = Dataset.from_dict({
    "question": questions,
    "answer": answers,
    "contexts": contexts,
    "ground_truth": ground_truths
})

result = evaluate(eval_dataset, metrics=[
    faithfulness, answer_relevancy,
    context_precision, context_recall
])
```

**产出**：一份评估报告，面试时说"faithfulness 达到 0.95，answer_relevancy 达到 0.92"。

### 步骤 1.3：多轮对话支持

**新建文件**：`backend/services/conversation_service.py`

**做什么**：支持追问，如"刚才那条再说详细点"。

**实现思路**：
```python
# 内存级会话管理（简单有效）
class ConversationService:
    def __init__(self, max_history=5):
        self.sessions = {}  # session_id -> [{"role": "user/assistant", "content": "..."}]
        self.max_history = max_history

    def get_history(self, session_id):
        return self.sessions.get(session_id, [])[-self.max_history*2:]

    def add_message(self, session_id, role, content):
        if session_id not in self.sessions:
            self.sessions[session_id] = []
        self.sessions[session_id].append({"role": role, "content": content})
```

**生成时拼接历史**：修改 `llm_service.py` 的 `_build_context()`，在 system prompt 后追加最近 5 轮对话。

**面试价值**：多轮对话是 RAG 系统的标配能力，补上这个缺口。

### 步骤 1.4：LLM 回答质量自评

**修改文件**：`backend/services/llm_service.py`

**做什么**：生成回答后，让 LLM 自己打分（0-10），分数低于阈值时重新生成。

```python
def self_evaluate(self, question, answer, search_results):
    prompt = f"""请评估以下回答的质量，只输出0-10的整数分数。

问题：{question}
检索到的参考资料：{[r['content'] for r in search_results]}
回答：{answer}

评分标准：
- 是否基于参考资料回答（不编造）
- 是否直接回答了问题
- 是否引用了来源

只输出分数。"""
    score = int(self.client.chat.completions.create(...).choices[0].message.content)
    return score >= 7  # 7分以下认为质量不足
```

**面试价值**：展示你对"输出质量保障"的思考，而且几乎零成本。

### 步骤 2.2：元数据过滤（Metadata Filtering）

**修改文件**：`backend/services/vector_store.py`

**做什么**：利用文档元数据（文档名、章节、条款类型）做预过滤。

```python
# 用户问"刑法相关" → 只在刑法文档中检索
# 用户问"违约赔偿" → 只检索民法典的合同编

def search_with_filter(self, question, top_k, doc_filter=None):
    where = None
    if doc_filter:
        where = {"source": {"$contains": doc_filter}}
    results = self.collection.query(query_texts=[question], n_results=top_k, where=where)
```

**面试价值**：元数据过滤是生产级 RAG 的必备能力，减少无关文档干扰。

### 步骤 2.3：父子文档检索（Parent-Child Retrieval）

**修改文件**：`backend/services/vector_store.py`

**做什么**：小片段用于精确匹配，命中后返回其所属的大段落（父文档）给 LLM。

```python
# 切分时同时存储：
# - child：按条款切分的小片段（用于检索）
# - parent：整个条款或整个章节（用于给 LLM）

# 检索流程：
# 1. 在 child 集合中检索 top-5
# 2. 通过 parent_id 找到对应的 parent 文档
# 3. 把 parent 文档给 LLM（上下文更完整）
```

**面试价值**：解决"检索粒度"和"生成需要上下文"之间的矛盾，RAG 高级技巧。

### 步骤 2.4：检索结果去重

**修改文件**：`backend/services/qa_service.py`

**做什么**：混合检索 + RRF 后，可能返回同一条款的不同片段，需要去重。

```python
def deduplicate_results(self, results):
    """基于文档名+条款号去重，保留分数最高的"""
    seen = set()
    unique = []
    for r in results:
        key = (r['source'], r.get('clause_number', ''))
        if key not in seen:
            seen.add(key)
            unique.append(r)
    return unique
```
